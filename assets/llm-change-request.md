# LLM 单次修改请求模板

本模板用于在嵌入式项目中提交一次代码修改请求。字段定义与 `.ai/LLM_RULES.md` 中的标准确认单保持一致，本文件仅做补充。

## 标准确认单

按 `.ai/LLM_RULES.md` 中「修改前强制确认」一节的核心 5 项填写：

```text
本轮目标：
Allowed Files：
最小验证方式：
硬件风险：
待确认问题：
```

未给出 Allowed Files 之前，不允许修改代码。

## 特殊场景补充（仅在适用时填写）

### 外设驱动

```text
完整信号路径（MCU 引脚 → 中间器件 → 终端器件）：
关键硬件参数（地址、片选、速率、模式、字节序、校验、复位/电源时序）：
```

未确认的环节视为未知参数，不得擅自假设。

### 修改 main / 主循环 / RTOS 任务入口

```text
临时屏蔽的调用及恢复条件：
新增调用的位置（在哪个区域、放在哪个调用之前/之后）：
```

只允许添加带注释的函数调用；复杂业务逻辑必须放到独立 `.c` / `.h` 文件中。

### 代码生成器项目（CubeMX / MCUXpresso 等）

```text
修改位置是否在 USER CODE 区域内：yes/no
如果 no：是否需要通过代码生成器重新生成：
```

## 长任务上下文检查（仅长任务或多阶段任务填写）

```text
是否切换了目标、硬件平台、外设、工程目录或 IDE：yes/no
是否存在多个历史需求可能混淆当前目标：yes/no
建议 clear / compact / 继续：
当前任务摘要（如建议 compact 时填写）：
```
